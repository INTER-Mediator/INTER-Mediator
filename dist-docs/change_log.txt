Ver.5.6 (in development)
- Well-designed CSS files as the 'default' theme are added by Yuko Oki. She is web designer and a lecturer of
  Kyoto University of Art and Design.
- Themes can be retrieved from accessing the definition file as below:
  def_file.php?theme=default&type=css&name=style.css
- The 'thosedays' theme are added for previous sample css style. The 'least' theme is just for pagenation,
  login panel and so on, it means least definitions css.
- In the Post Only mode, radio buttons don't require to set the name attribute. The nodes having the
  same data-im attribute automatically set the appropriate name attributes.
- New interleave method INTERMediatorOnPage.modifyPostOnlyContext is defined. The parameter is post only
  context definition and should return a context definition which is modified or not.
- For preventing to update by editing element's value, data-im-control="unbind" is introduced. This attribute
  causes just assign the field value to checkbox, but it doesn't happen to update with clicking it.
- Added titles to almost all sample HTML files.
- Add "CodeMirror" (CodeMirror 5.23.0) in "INTER-Mediator-Support" directory for INTER-Mediator-Server VM.
- Update Vagrantfile, deploy.sh, recipe.rb and sample_spec.rb for changing OS of INTER-Mediator-Server VM.
- The access privilege of directories and file of uploaded file set to 755. Previously it was 744, but
  a rental server which is limited feature tries to prevent to set all bits of group/other.
- Updating and Creating record operations always return the records affected with its operations.
- JSDoc generating script is added as dist-docs/jsdocgen.sh. You have to install JSDoc with npm.
  Of course, all JS codes aren't documented so far, we will fill up step by step.
- The $altThemePath variable in params.php file can have the alternative theme directory path for adding
  solution specific theme.
- The $themeName variable in params.php file can have the default theme name for all web pages.
- JavaScript component "jsonformater" is added. Useage is: <div data-im-widget="jsonformater" data-im="t@f"></div>
- The style named 'thosedays' applies CSS definitions as same as older version's sample files.
- Checking the result of voting for target specifications, and show error message if the context which is specified
  in target specifications in page file doesn't exist.

Ver.5.4 (August 24, 2016)
- You don't have to call INTERMediator.construct() anymore. If you want to do anything before page
  generating, describe "INTERMediatorOnPage.doBeforeConstruct = function(){..};" on any valid place.
  The node having the id attribute "nonsupportmessage" will disappear after page generating. If you
  require to the another id value, you can set it into params.php as the value "$nonSupportMessageId".
- "IN" operator in SQL db can be accepted with DB_PDO class. The value should be an array of itemized
  values.
- If the automatic calling to construct method doesn't requre, descirbe the below in doBeforeConstruct
  or prior to that: INTERMediatorOnPage.isAutoConstruct = false;
- Creating Cross Table for PDO. It contains 3 contexts for column, row and value cells.
  See the sample Samples/Practices/crosstable.html.
- The "post-query-stored" key in context of the definition file is introduced. It will be specified
  a method name, ant the method has to be define in the INTERMediatorOnPage object with one variable
  for referring context object. It will be called after query and stored values in context.
- The second "option" parameter of IM_Entry function can have the key "local-context" as the array of
  "key" and "value" keyed elements. These affect as the initial values of relevant local context value.
- The variable "$valuesForLocalContext" in params.php supplies initial values for local context.
  The value of this variable has to be an associated array with key and value.
- Improve setting search criteria when using FileMaker Server 12 or later to use the same fields for
  multiple values in the same "and" or "or" search criteria. (Use "-findquery" command internnally,
  but compound find requests using both "and" and "or" search criteria is still not supported.)
- Support the script parameters for FileMaker Server (controlled by 'parameter' key of 'script' key).
- FileUploader can detect exceeding file size limit and returns valid error messages.
- New Sample "Sample_API" is added. This is for explanation of server-side apis.
- On the Master-Detail style UI, the scrolled point in Master display can be retrieved when the page
  is backed from the Detail display.
- Add "CWPKit" in "lib" directory for FileMaker Server to remove a dependency of FX.php in the future.
- Customizing X-Frame-Options and Content-Security-Policy headers in the params.php file.
- Customizing the file path format of uploaded file with params.php file.
- The IMLibUI.valueChange method is locked with the id attribute value to prevent the error when multiple
  updating processes run simultaneously.
- Supporting jQuery File Upload. Check the file Samples/Sample_website/fileupload_jQuery_MySQL.html.
- Reduce error messages when the mbstring extension is disabled.
- Add .eslintrc.js.
- Update sample_schema_mysql.txt, dbupdate.sh, recipe.rb and sample_spec.rb for the INTER-Mediator-Server
  virtual machine.
- The 'choice', 'condition', 'accumulate' and 'lastindexof' functions are introduced into expression.
- On Post-Only mode, nodes with the same data-im-group attribute are corrected as the value in one field.
- The ignorePost method is introduced in DB_Proxy class for ignoring the $_POST values in adviser class.
